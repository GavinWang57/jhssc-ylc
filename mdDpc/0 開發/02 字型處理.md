# 字型處理指南

本文件說明如何在專案中使用 Noto Sans TC（思源黑體）字型。

## 1. 方法一：使用 Google Fonts CDN（在 index.html 中載入）

這是最簡單快速的方法，適合快速原型開發或不想增加專案打包大小的情況。

### 1.1 在 index.html 中加入字型連結

在 `index.html` 的 `<head>` 區塊中加入以下程式碼：

```html
<!doctype html>
<html lang="zh-TW">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <!-- Google Fonts - Noto Sans TC -->
    <link rel="preconnect" href="https://fonts.googleapis.com" />
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
    <link
      href="https://fonts.googleapis.com/css2?family=Noto+Sans+TC:wght@300;400;500;700&display=swap"
      rel="stylesheet"
    />

    <title>jhssc-ylc</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>
```

### 1.2 說明

- **`preconnect`**：提前建立與 Google Fonts 伺服器的連線，加快字型載入速度
- **字重參數** `wght@300;400;500;700`：
  - `300` - Light（輕）
  - `400` - Regular（標準）
  - `500` - Medium（中等）
  - `700` - Bold（粗體）
- **`display=swap`**：先顯示系統預設字型，等 Google 字型載入完成後再切換，避免文字閃爍（FOIT）

### 1.3 優缺點

**優點：**

- ✅ 簡單快速，不需要安裝套件
- ✅ Google CDN 自動優化並提供快取
- ✅ 不增加專案打包大小

**缺點：**

- ❌ 需要網路連線才能載入字型
- ❌ 首次載入時可能會有延遲
- ❌ 受限於 Google Fonts API 的可用性

## 2. 方法二：使用 npm 套件（本地安裝）

### 2.1 安裝字型套件

打開終端機執行：

```bash
npm install @fontsource/noto-sans-tc
```

## 3. 在專案中引入字型（方法二的後續步驟）

在 `main.jsx` (或 `main.tsx`) 入口檔案**最上方**加入以下引入語句：

```javascript
import "@fontsource/noto-sans-tc"; // 預設載入 400 字重 (Regular)
import "@fontsource/noto-sans-tc/500.css"; // 載入 500 字重 (Medium) 供標題使用
import "@fontsource/noto-sans-tc/700.css"; // 載入 700 字重 (Bold)
```

### 2.1 在單個 JSX 檔案中引入

如果只想在特定元件中使用字型，可以在該 JSX 檔案頂部引入：

```jsx
// 在元件檔案開頭引入
import "@fontsource/noto-sans-tc/400.css"; // Regular
import "@fontsource/noto-sans-tc/500.css"; // Medium

function MyComponent() {
  return (
    <div style={{ fontFamily: '"Noto Sans TC", sans-serif' }}>
      <h1 style={{ fontWeight: 500 }}>標題使用 Medium</h1>
      <p style={{ fontWeight: 400 }}>內文使用 Regular</p>
    </div>
  );
}

export default MyComponent;
```

或者使用 CSS Module 或 styled-components：

```jsx
// 使用 CSS Module
import styles from "./MyComponent.module.css";
import "@fontsource/noto-sans-tc";

function MyComponent() {
  return (
    <div className={styles.container}>
      <h1 className={styles.title}>標題</h1>
      <p className={styles.content}>內文</p>
    </div>
  );
}
```

```css
/* MyComponent.module.css */
.container {
  font-family: "Noto Sans TC", sans-serif;
}

.title {
  font-weight: 500;
}

.content {
  font-weight: 400;
}
```

> **注意：** 建議在 `main.jsx` 中全域引入字型，這樣可以：
>
> - 避免重複載入相同字型檔案
> - 確保整個應用程式字型風格一致
> - 減少打包後的檔案大小

## 4. CSS 基礎設定

### 4.1 全域字型設定

```css
body {
  font-family: "Noto Sans TC", sans-serif;
}
```

### 4.2 專案表格字型設定

```css
/* 專案表格通用字型設定 */
.project-table {
  font-family:
    "Noto Sans TC",
    -apple-system,
    BlinkMacSystemFont,
    "Segoe UI",
    Roboto,
    "Helvetica Neue",
    Arial,
    sans-serif;
  -webkit-font-smoothing: antialiased; /* 讓字體在 Mac 上更細膩 */
}
```

## 5. 核心技巧：開啟等寬數字

專案管理最常遇到進度百分比（如 11% 與 88%），若寬度不同，表格視覺會顯得左右晃動。  
Noto Sans TC 內建支援「等寬數字」功能，請務必在數字欄位開啟：

```css
.table-number {
  /* 強制讓數字像格點一樣對齊，不會因為 1 和 8 寬度不同而跳動 */
  font-variant-numeric: tabular-nums;
  text-align: right; /* 數字欄位建議一律靠右 */
}
```

## 6. 字重（Weight）的階層應用

利用 Noto Sans TC 豐富的字重來優化資訊層級：

| 元素類型               | 建議字重                   | 字級與顏色建議                           |
| ---------------------- | -------------------------- | ---------------------------------------- |
| **表格標題 (th)**      | Medium (500) 或 Bold (700) | 字級略小（如 13px），顏色調淡（如 #666） |
| **專案名稱**           | Regular (400)              | 顏色最深（如 #222）                      |
| **輔助數據（如日期）** | Light (300)                | 增加與主標題的對比                       |

### 範例 CSS

```css
.project-table th {
  font-weight: 500; /* Medium */
  font-size: 13px;
  color: #666;
}

.project-table .project-name {
  font-weight: 400; /* Regular */
  color: #222;
}

.project-table .date {
  font-weight: 300; /* Light */
  color: #888;
}
```

## 7. 最佳實踐建議

- ✅ 僅載入專案實際使用的字重，避免增加不必要的檔案大小
- ✅ 數字欄位統一使用 `font-variant-numeric: tabular-nums`
- ✅ 使用字重來建立視覺層級，而非過度依賴顏色或字級變化
- ✅ 在 Mac 環境下啟用 `-webkit-font-smoothing: antialiased` 改善顯示效果
